================================================================================
  SALES SUMMARY AUTOMATION
================================================================================

Python-скрипт автоматически строит сводный отчёт по продажам из Excel-файла
с помесячными данными.

--------------------------------------------------------------------------------
ФАЙЛЫ
--------------------------------------------------------------------------------

  create_sales_summary.py   — основной скрипт
  sales_report.xlsx         — исходный файл с продажами (вход)
  sales_summary.xlsx        — готовый сводный отчёт (выход)
  requirements.txt          — зависимости Python
  README.txt                — эта инструкция

--------------------------------------------------------------------------------
ЧТО НУЖНО УСТАНОВИТЬ
--------------------------------------------------------------------------------

  1. Python 3.10+ (рекомендуется 3.11 / 3.12 / 3.13)

  2. Зависимости:

        pip install -r requirements.txt

     или:

        pip install pandas openpyxl

--------------------------------------------------------------------------------
КАК ЗАПУСТИТЬ
--------------------------------------------------------------------------------

  В папке проекта:

        python create_sales_summary.py

  Это прочитает sales_report.xlsx и создаст / перезапишет sales_summary.xlsx.

  Свои пути:

        python create_sales_summary.py path\to\input.xlsx
        python create_sales_summary.py path\to\input.xlsx -o path\to\output.xlsx

  Пример:

        python create_sales_summary.py new_month.xlsx -o sales_summary.xlsx

--------------------------------------------------------------------------------
ФОРМАТ ИСХОДНОГО ФАЙЛА (sales_report.xlsx)
--------------------------------------------------------------------------------

  Скрипт читает лист "Sales Data", если он есть, иначе — первый лист.

  Обязательные колонки (названия могут немного отличаться):

    Region / Area / Territory
    Product Category / Category
    Product Name / Product / Item
    Revenue / Sales / Amount / Total

  Желательные колонки:

    Date          — для помесячного роста (надёжнее, чем колонка Month)
    Quantity      — для Top 10 (кол-во проданных единиц)

  Допустимые алиасы заголовков обрабатываются автоматически
  (регистр и лишние пробелы не важны).

  Как подставить новый месяц:
    1. Положи новый файл с той же структурой колонок
       (например, замени sales_report.xlsx или укажи путь в команде)
    2. Запусти скрипт снова
    3. Открой обновлённый sales_summary.xlsx

--------------------------------------------------------------------------------
ЧТО ВНУТРИ sales_summary.xlsx
--------------------------------------------------------------------------------

  Overview
    — KPI: Total Revenue, Transactions, Regions, Categories, Products
    — Key insight (рост/падение последнего месяца)
    — Таблицы Sales by Region и Sales by Category (+ Share %)
    — 2 столбчатые диаграммы

  Sales by Region
    — Полная таблица + бар-чарт

  Sales by Category
    — Полная таблица + бар-чарт

  Top 10 Products
    — Топ-10 по выручке (Rank, Quantity, Total Sales, Share %)
    — Бар-чарт; топ-3 подсвечены

  Monthly Growth
    — Total Sales, Previous Sales, Change ($), Growth %
    — Growth % уже посчитан (не формула): зелёный = рост, красный = падение
    — Линейный тренд продаж + столбцы Growth %

--------------------------------------------------------------------------------
ФОРМУЛА РОСТА
--------------------------------------------------------------------------------

  Growth % = (Продажи текущего месяца − Продажи прошлого месяца)
             / Продажи прошлого месяца

  Первый месяц в данных — без сравнения (ставится "—").

--------------------------------------------------------------------------------
ФОРМАТИРОВАНИЕ
--------------------------------------------------------------------------------

  • Жирные заголовки на тёмно-синем фоне
  • Валюта:  $#,##0.00
  • Проценты: 0.0%
  • Зебра-строки, totals, KPI-карточки
  • Цвета для positive / negative growth

--------------------------------------------------------------------------------
ЧАСТЫЕ ВОПРОСЫ
--------------------------------------------------------------------------------

  Q: Скрипт пишет Error: Input file not found
  A: Проверь, что sales_report.xlsx лежит в той же папке, откуда запускаешь
     команду, или укажи полный путь к файлу.

  Q: Missing required columns...
  A: В файле нет одной из колонок: Region, Category, Product, Revenue
     (или их синонимов). Переименуй заголовки или добавь колонку.

  Q: Growth % пустой / n/a
  A: Нужны данные минимум за 2 разных месяца и колонка Date
     (или корректные даты для группировки по месяцам).

  Q: Перезаписал sales_summary.xlsx — пропали мои правки
  A: Скрипт каждый раз создаёт файл заново. Не правь summary вручную —
     меняй исходник и запускай скрипт снова.
